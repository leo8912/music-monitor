# 项目待办事项清单 (Project Roadmap)

> 更新时间: 2026-01-26
> 目标: 完善 Music Monitor 全栈音乐监控与播放系统

## 🅿️ Phase 1: 核心体验完善 (High Priority)
*解决当前最影响用户体验和核心流程缺失的问题*

### [Frontend] 前端开发
- [ ] **艺人详情页 (Artist Detail)**
    - [ ] 桌面端: 点击 "现在就听" 或 "资料库" 中的艺人头像，跳转至详情页 (展示热门歌曲、专辑列表、相似艺人)。
    - [ ] 移动端: 对应的移动端视图适配。
    - *分析*: 当前点击艺人无反应或仅播放，用户无法查看该艺人更详细的收藏/作品。
- [ ] **专辑详情页 (Album Detail)**
    - [ ] 展示专辑封面、发行日期、完整曲目列表。
    - [ ] 支持整张专辑播放/收藏。
- [ ] **搜索体验增强**
    - [ ] 搜索结果分类展示 (单曲、艺人、专辑)。
    - [ ] 支持本地资料库搜索 (目前似乎只搜在线?)。
- [ ] **播放列表/队列管理**
    - [ ] 查看当前播放队列 (Up Next)。
    - [ ] 简单的拖拽排序或移除歌曲。

### [Backend] 后端开发
- [ ] **元数据补全优化**
    - [ ] 确保下载后自动嵌入封面、歌词、Artist、Album 等 ID3 信息 (检查 `metadata_service.py` 覆盖率)。
- [ ] **下载队列管理**
    - [ ] 处理下载失败的任务 (重试机制)。
    - [ ] 防止重复添加到下载队列 (去重逻辑复查)。

---

## Ⓜ️ Phase 2: 功能扩展 (Medium Priority)
*增强系统的可用性和丰富度*

### [Frontend] 前端开发
- [ ] **歌词显示与滚动**
    - [ ] 桌面端/移动端播放器界面支持实时歌词滚动 (LRC解析)。
- [ ] **自定义歌单 (Playlists)**
    - [ ] 创建、重命名、删除自定义歌单。
    - [ ] 将歌曲添加到特定歌单。
- [ ] **批量操作**
    - [ ] 资料库/历史记录支持多选，通过 Shift/Ctrl 批量删除或下载。
- [ ] **系统状态仪表盘**
    - [ ] 在设置页或独立页面展示：磁盘占用、近期下载成功率、API 调用统计。

### [Backend] 后端开发
- [ ] **多用户/权限系统 (可选)**
    - [ ] 如果计划多人使用，需要区分用户收藏和播放历史。
- [ ] **智能清理**
    - [ ] 自动清理超过 N 天的未收藏/未播放缓存文件。
